# The Browser Shell Gets a Skeleton

**Rust shell rebuild · Buster Claw 0.1.0**

Dismantling the 2,526-line `browser.rs` monolith and closing the 150-to-8 test asymmetry — without renaming a command, moving a pixel, or touching a single contract the Elixir side depends on.

> Researched 2026-07-21. Ground truth read from `desktop/tauri/src/*` at HEAD, the full caller map (chrome.js, browser.js hooks, tab_strip.js, `Browser.Bridge`/`Capture`, `Commands.Web` decoders, browser controllers), both capability files, build.rs, ci.yml, and the archived 07-18 agent-web-automation roadmap. Every contract listed below was traced to a live call site, not assumed.

---

## The short version

**`browser.rs` is the most dangerous file in the repo.** It holds 33 Tauri commands, all browser state, every injected-JS builder, and all the objc/WKWebView FFI in one 2,526-line file. It has 8 tests. The Elixir side of the same feature has dozens of test files and a CI gate; the Rust side has neither — CI *compiles* Rust on release tags and never runs `cargo test`, clippy, or fmt. The code that executes agent-supplied selectors inside real webviews is the least-verified code we ship, baked into the artifact that is slowest to fix.

**The failure mode is already on the record.** On 07-17 we discovered the co-presence commands had been ACL-dead in the packaged app since Phase 3 — registered in `generate_handler!` but missing from `build.rs`/capabilities. Dev builds masked it. Nothing failed; the feature just silently didn't exist. That class of bug is structural to this layer, and no runtime test we have can catch it cheaply.

**And it isn't done recurring: `speak`/`stop_speaking` are missing from `build.rs` right now.** They're in `generate_handler!` and allowed in `capabilities/default.json`, but absent from the `commands(&[...])` manifest — voice works today only because a stale `permissions/autogenerated/speak.toml` is still sitting on disk from an earlier build. A regenerate on a clean checkout could kill TTS in the packaged app. Found while writing this document; fixed in Phase 0.

**The fix is not a rewrite.** The monolith already contains a clean seam: a pure layer (JS builders, the `js_str` escaper, `ActTarget` resolution, geometry, label/sid handling, all of `BrowserState`'s bookkeeping) that never touches Tauri, and an impure shell (webview lifecycle, four objc FFI functions) that can't be meaningfully faked. So: **functional core, imperative shell**. Everything that *decides* becomes a pure, tested function; everything that *touches WebKit* becomes a dumb loop. Same pattern as the Elixir `FlowRunner` with its injected `exec:`/`screenshot:` — translated to Rust without trait machinery.

**Six phases, each shippable, contracts frozen throughout.** Tests are written against the monolith *first* (characterization), CI starts gating *second*, and only then does code move — in `--color-moved`-reviewable, move-only commits. The commands never leave `browser/mod.rs`, so `main.rs`, `build.rs`, and the capability files are untouched by the refactor and a July-17 recurrence is structurally impossible mid-flight. The capstone is the one thing LEFTOVERS still lists as outstanding: a real packaged-app smoke test.

## 1. What we're rebuilding, and what we are not

The feature's architecture is sound and stays exactly as it is: native chrome webview + per-tab content webviews, the Bridge/Capture round-trip through Phoenix, sandbox-by-default flows, the WKUIDelegate ceiling — all accepted, all unchanged. The UI does not move. No JS file, no Elixir file, no template changes in any phase.

What changes is the *shape* of the Rust: one file becomes eight, ~40 named tests pin the behavior, and three gates (local script, precommit, CI) make the pins permanent.

### The monolith's anatomy (as it stands)

| Band | Contents | Testable today? |
|---|---|---|
| Commands (L434–1706) | 33 `#[tauri::command]` fns, all `Result<_, String>` | No — thin-ish, but tangled with everything below |
| Pure JS core (L1012–1425, L2367) | `js_str`, `find_elements_js`, `click_js`, `fill_js`, `wait_probe_js`, `extract_matches_js`, `error_data`, `ActTarget` | **Yes — 8 tests exist** (L2390) |
| State (L80–340) | `BrowserState`: 8 Mutex/Atomic fields, ~25 bookkeeping methods, zero Tauri deps | Yes, but untested |
| Orchestration (L1905–2365) | `ensure_chrome`, `create_content`, `enforce_tab_budget`, `tab_eval`, history/download POSTs | Only via the Elixir fake-desktop suite |
| FFI (L1453–1904) | `eval_with_result`, `apply_content_blocking`, `capture_webview`, `webview_title` — objc, macOS-gated, with non-mac stubs | No, and never will be headlessly |

## 2. The invariants — frozen for the duration

Stated once, verified every phase. Violating any of these is a rollback, not a judgment call.

1. **Command names and camelCase JS arg names are byte-identical.** `surfaceId`, `tabId`, `timeoutMs`, `waitMs`, `downloadId`, `chromeUrl`, `contentUrl`, `factor`, `backwards`, `query`, `selector`, `text`, `value`, `index`, `condition`, `session`, `path`, `enabled`, `collapsed`, `url`. Tauri's camel→snake mapping makes every Rust param name a JS contract.
2. **serde return structs keep their field names.** `EvalData{data}`, `ReadPage{data}`, `Screenshot{data,url}`, `CurrentTab{url,title}` — read by name in JS and re-decoded in `commands/web.ex`. The inner `.data` JSON schemas too: `{ok,matched,waited_ms,condition}`, `{ok,label,matched_by}`, `{url,title,text,links}`, `{matches,count}`.
3. **`Option` stays `Option`.** `surface_id` on every co-presence command and on `browser_close` (tab_strip.js calls it with no args); `index`/`selector`/`text` on click/fill — `index: 0` is valid, JS sends `?? null`.
4. **Async stays async** on every eval/snapshot command (main-thread deadlock contract). `wait` returning `matched:false` is a *success*. `browser_find_count` returns a bare number.
5. **`main.rs`, `build.rs`, `capabilities/*.json` are untouched by refactor phases.** Phase 0 touches `build.rs` exactly once — the voice fix — as its own commit. No renames, anywhere, ever, in this roadmap.
6. **The `sanitize_sid` parity fixture stays in lockstep** with `browser_chrome_controller_test.exs` (the binding comments on both sides survive the move).

## 3. The target shape

`browser.rs` becomes `src/browser/`, and the commands **stay in `browser/mod.rs`**. This is the load-bearing decision: `generate_handler![browser::browser_open, ...]` resolves both the fn and the hidden `__cmd__*` macro item at that exact path, so keeping commands at `browser::*` means the registration triangle (`main.rs` / `build.rs` / capabilities) never opens during the rebuild.

```
desktop/tauri/src/
├── main.rs                    unchanged
├── browser/
│   ├── mod.rs        ~550    33 thin command wrappers · serde return structs
│   │                         handle_menu_shortcut · active_sid/active_content
│   ├── js.rs         ~480    THE SECURITY CORE — js_str, error_data, every *_js
│   │                         builder, injected consts, ActTarget (+resolve_js)
│   ├── labels.rs      ~90    sanitize_sid (Elixir parity) · chrome/content_label
│   │                         parse_web_url
│   ├── geometry.rs    ~70    content_box · layout consts
│   ├── state.rs      ~330    BrowserState · DownloadLog · bookkeeping methods
│   │                         NEW: pure select_evictions(mru, live, active)
│   ├── webviews.rs   ~480    impure orchestration — ensure_chrome, create_content,
│   │                         show/place, enforce_tab_budget (loop over
│   │                         select_evictions), tab_eval, label iteration
│   ├── ffi.rs        ~380    objc + stubs — eval_with_result, content blocking
│   │                         (include_str! path → ../blocklist.json),
│   │                         capture_webview, webview_title
│   └── notify.rs     ~220    reqwest → /browser/history, /browser/download ·
│                             emit_navigated · ping_agent_activity
├── terminal.rs               unchanged + tests for pure helpers
├── voice.rs                  unchanged (111 lines — no split)
└── workspace.rs              unchanged — it was the pattern all along
tests/
└── acl_lockstep.rs   ~150    the July-17 guard (§4)
```

Submodule items are `pub(crate)`, never `pub` — the crate's public surface remains exactly the command set.

**What we deliberately do not build:**

- **No cargo workspace / separate `browser-core` crate.** It would perturb `tauri-build` and the `rust-cache` config for one guarantee — "the core has no Tauri dependency" — that a 5-line test asserts for free (`pure_modules_have_no_tauri_imports` greps the four pure files for `use tauri`).
- **No `WebviewPort` trait, no mock webviews.** `tauri::Webview` isn't headlessly constructible; a fake would test the fake. Instead every branch worth testing moves *out* of the shell into pure functions, and `webviews.rs` ends up too dumb to be wrong in interesting ways. The Elixir fake-desktop suite already covers the JSON boundary from above.
- **No touching terminal/voice structure.** 311 and 111 lines are not monoliths. They get tests, not surgery.

## 4. The test regime (~40 named cases)

**`js.rs` — injection safety.** The existing 8 migrate; added: `js_str` escapes U+2028/2029 and `</script>`; hostile query/selector/value strings (`"; alert(1);//`) stay inert string literals in `find_elements_js`/`click_js`/`fill_js`; `act_target_priority_selector_over_text_over_index`; `act_target_index_zero_is_valid`; `wait_probe_js_rejects_unknown_condition`. This is the layer where a bug is an XSS-into-`__TAURI__`-adjacent incident, so it gets the densest coverage.

**`labels.rs` / `geometry.rs`.** `sanitize_sid_parity_fixture` (same table as the Elixir controller test); `parse_web_url_accepts_http_https_rejects_file_javascript`; `content_box_expanded` / `content_box_collapsed`.

**`state.rs` — state-machine tests.** `BrowserState::default()` is constructible with no runtime, so: MRU front-on-switch; `select_evictions_never_evicts_active` / `prefers_lru` / `respects_max_live_tabs`; ephemeral-tab budget behavior (frozen as *observed*, read line-by-line — not as assumed); close-tab clears active/MRU/ephemeral; download ids monotonic; per-surface boxes; sidebar/shown roundtrips.

**`mod.rs` — contract snapshots.** Serde field names of all four return structs; the wait JSON incl. `matched:false` serializing as success; `error_data`/`NO_TARGET_DATA` shapes; find_count's bare number. These tests are the Rust half of what `web.ex`'s decoders assert from the other side.

**`tests/acl_lockstep.rs` — the July-17 guard.** `include_str!` on `build.rs` and `main.rs`, `serde_json` on both capability files; then three subset assertions: every `generate_handler!` command is in `build.rs`'s manifest; every manifest command is allowed by some capability; no capability allows an orphan. Minimum-count tripwires (`>= 30`) make silent parser under-extraction impossible. This test fails on today's tree — that's the voice finding — and from Phase 1 on, it fails in CI before a dead-in-packaged-app command can ever ship again.

## 5. The gates

- **CI (`ci.yml`)** gains a `rust` job on `macos-14` — the objc path *is* the shipped path; ubuntu would test the stubs — running `cargo fmt --check`, `cargo clippy --all-targets -- -D warnings`, `cargo test`, warmed by `swatinem/rust-cache` (`workspaces: desktop/tauri`, same as release-desktop.yml; warm runs ≈2–3 min). And a `js` job on ubuntu running `bun test assets/js`, because the bun tests existing-but-unenforced is the same disease in a second language.
- **`desktop/tauri/rust-toolchain.toml`** pins the channel + rustfmt/clippy components — `.tool-versions`, translated.
- **`scripts/check_rust.sh`** mirrors the CI steps exactly and gets appended to the `precommit` alias, so `ci.yml`, precommit, and the docs describe one artifact.
- **`docs/QUALITY.md`** Tauri section upgrades from `cargo check` to the script.

## 6. The phases

Every phase leaves `cargo tauri dev` fully functional. Move phases are move-only commits, reviewed with `git diff --color-moved=dimmed-zebra`.

| Phase | What ships | Size |
|---|---|---|
| **0 · Characterization freeze** | `serde_json` dev-dep, toolchain file, `tests/acl_lockstep.rs`, and every contract/serde/js-builder test — written against the monolith **as-is**. Fix the voice/`build.rs` registration (own commit). Size and settle clippy/fmt on existing code (targeted `#[allow]` for objc blocks, no FFI rewrites). | +~600 test |
| **1 · CI gate** | `rust` + `js` jobs, `check_rust.sh`, precommit wiring, QUALITY.md. Deliberately break one capability entry on a scratch branch to watch the ACL test go red. | +~170 |
| **2 · Pure core out** | `browser.rs` → `browser/`; extract `js.rs`, `labels.rs`, `geometry.rs`; tests migrate into their modules. Phase-0 tests pass unmodified — they froze behavior, so the move can't change it. | ~700 moved |
| **3 · State out** | Extract `state.rs`; introduce `select_evictions`; `enforce_tab_budget` becomes a loop over it; 10 state-machine tests land. | ~350 moved, +~250 |
| **4 · Shell split** | Extract `ffi.rs`, `notify.rs`, `webviews.rs`. `mod.rs` shrinks to wrappers + structs. `browser.rs` the file ceases to exist. | ~1,100 moved |
| **5 · Siblings + packaged smoke** | Terminal pure-helper tests. `scripts/smoke_desktop.sh`: build the packaged app, launch it, wait on `/_health`, drive `browser_open → navigate → read → click → screenshot → close` through the Phoenix HTTP bridge, assert shapes. Pre-release, not per-PR. Closes the LEFTOVERS smoke item — the dynamic half of July-17 prevention, where the ACL test is the static half. | +~200 |

Net: ~2,150 lines moved, ~1,050 added (all tests/CI/scripts), **0 lines of contract changed**.

## 7. Verification matrix

- **Every phase:** `check_rust.sh` + `mix test` (the fake-desktop suite proving every Elixir decoder still parses every payload) + move-only diff review.
- **Phases 2–4 additionally:** `scripts/smoke_command_surface.sh`, plus the manual `./scripts/dev.sh` checklist — surface open; 7-tab churn (eviction); click/fill/wait/extract flow; screenshot; download + reveal; sidebar collapse; zoom; find-bar count; popup-as-tab; menu shortcuts.
- **Phase 5:** `smoke_desktop.sh` against the built DMG's app — the only check that exercises production ACL resolution end-to-end.

## 8. Risk register

| Risk | Odds | Containment |
|---|---|---|
| `generate_handler!` path breakage from moving commands | — | Eliminated by design: commands never leave `browser/mod.rs` |
| clippy `-D warnings` churn on 3,778 legacy lines | Med | Sized first in Phase 0; targeted `#[allow]` with comments on objc blocks |
| Behavior drift hiding inside a "pure move" | Med | Tests exist *before* any move; move-only commits; `--color-moved` review |
| Eviction/ephemeral semantics mischaracterized | Med | Phase-0 tests assert line-by-line *observed* behavior |
| `include_str!("blocklist.json")` path break in `ffi.rs` | High, trivial | Compile error; path becomes `../blocklist.json` |
| ACL guard parser rots with formatting changes | Low | Minimum-count tripwires fail loudly |
| macOS CI minutes | Low | rust-cache + existing concurrency-cancel; bun job stays on ubuntu |
